正解がない中で、マニュアルや仕様書をゼロから作ってきました。

業務整理 マニュアル 仕様書 仕組み化 自動化

　仕事のスタンス

  • フォーマットや前例がなくても、まずは形にしてみる
  • 使われない資料より、「見ればわかる」ことを優先する
  • 業務を止めないことを最優先に考える

　実務・事例

業務を止めないための工夫と仕組み化

仕様書・マニュアル作成

既存のフォーマットや前例がない中で、Excelと画面キャプチャを使いながら、 実際の操作画面に沿った仕様書・マニュアルを作成してきました。

スクリーンショットを多用し、「見れば分かる」状態になることを意識して構成しています。

マニュアル・資料作成（Canva）

文字だけでは伝わりにくい部分については、Canvaを使ってレイアウトを整え、 視覚的に理解しやすい資料を作成しました。

デザイン性よりも、業務で実際に使われることを重視しています。

業務サポート用ツール作成（Python）

自社サービスにCSV出力機能がなかったため、必要な情報を取得できるよう、 Pythonを使ってデータを取得・CSV出力する簡易ツールを作成しました。

既存の業務フローを止めず、手作業を減らすことを目的とした対応です。

運用前提の名刺フォーマット設計（Canva）

会社の名刺デザインに一定のルールがある中で、 Canvaを使って共通フォーマットを設計しました。

所属・名前・電話番号・メールアドレスなどの情報を 差し替えるだけで各社員分を作成できる構成とし、 量産・更新・発注までをスムーズに行えるようにしています。

一度作って終わりではなく、 情報が変わっても業務が止まらない形を意識しました。

その場で必要なものを、業務を止めずに形にすることを大切にしています。

　スキル・道具

業務で使ってきた道具

Excel / Canva / Python / VSCode / ChatGPT

得意なこと

  • 画面キャプチャや図で伝わる形に整理
  • 前例がなくてもまず叩き台を作る
  • 運用を止めない現実的な落とし所

　プロセス

  1. 状況や目的を簡単にヒアリングする
  2. 前例や既存資料があれば確認する
  3. まずは簡易な形で全体像を作る
  4. 画面キャプチャや図を使ってわかりやすく整理する
  5. 実際の業務で使えるかを意識して調整する

　これから

これまでと同様に、業務を止めないことを大切にしながら、 マニュアル作成や業務整理、仕組み化を続けていきたいと考えています。

今後は、自動化やテンプレ化など、手作業を減らす工夫にも取り組んでいきたいです。
